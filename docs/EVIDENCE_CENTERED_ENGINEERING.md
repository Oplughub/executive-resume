# Evidence-Centered Engineering

Evidence-Centered Engineering is a design philosophy for building systems where claims, outputs, and decisions remain connected to the evidence that supports them.

It favors traceability over persuasion, reproducibility over performance theater, and durable records over memory.

## Definition

Evidence-Centered Engineering is the practice of designing systems so that meaningful claims can be traced back to preserved evidence, reviewed by humans, improved over time, and published in forms that remain useful to both people and machines.

## Core Principles

1. Evidence before claims.
2. Truthful metrics only.
3. Structured records before polished presentation.
4. Human judgment remains accountable.
5. Presentation layers are replaceable.
6. Version history matters.
7. The system should make future review easier, not harder.

## Why It Matters

AI systems can generate language quickly.

That makes evidence more important, not less important.

When a system produces a resume, interpretation, report, summary, recommendation, or decision-support output, the question should not only be:

> Does this sound good?

The stronger question is:

> What evidence supports this, and can we inspect the path from evidence to output?

## How This Repository Applies It

This repository applies Evidence-Centered Engineering to professional documentation.

The resume is not treated as the source of truth.

The structured professional record is the source of truth.

The resume, PDF, HTML page, portfolio, biography, or CV are generated publications.

```text
meaningful work
    -> preserved evidence
    -> structured professional record
    -> version history
    -> generated publication
```

## Practical Test

Before adding a claim to a resume, report, publication, or system output, ask:

1. What evidence supports this?
2. Where is that evidence preserved?
3. Can I explain this honestly in conversation?
4. Can this be updated later without rewriting the whole system?
5. Does the presentation make the evidence easier to trust?

If the answer is yes, the claim belongs.

If the answer is no, preserve the work elsewhere until the evidence is stronger.
