# Professional Provenance

Professional Provenance is the version-controlled history of meaningful professional evidence from which resumes, portfolios, biographies, CVs, and public profiles can be generated.

## Core Idea

A resume should not be the place where you remember your career.

A resume should be generated from a professional record that is maintained while the work is still fresh.

## Why It Matters

Many people now use AI systems to draft, plan, analyze, summarize, code, design, and publish work.

Useful work can end up scattered across chat history, downloaded files, screenshots, notes, and email drafts.

A Git repository gives that work a durable place to live.

## The Pattern

```text
meaningful work
    -> preserved evidence
    -> structured professional record
    -> generated publications
    -> resume / portfolio / CV / biography / website
```

The resume is one output.

The professional record is the source.

## The Role of AI

AI can help organize, draft, revise, summarize, and explain.

AI can also help people learn Git, HTML, CSS, and publishing workflows.

The human remains responsible for accuracy, evidence, judgment, and final representation.

## The Role of Git

Git gives a folder memory.

A commit is a saved checkpoint.

A repository is a record of how work changed over time.

That record can be useful even when the work is not software.

## The Rule

Optimize for truth, reproducibility, and clarity over visual novelty.

If the number is 998, write 998.

If the milestone happens later, commit it later.

If the evidence does not exist yet, do not claim it yet.
